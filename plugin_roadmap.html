1. 자동화

솔루션개발부(iOS/Android) 단독 작업
코드 작성 · 단위 테스트 · UI 테스트

근본 원인
⚠ Root Cause

"코드 작성과 검증을 전적으로 개발자 개인의 역량과 판단에 의존"
자동화 장치가 없어 코드 품질이 개발자마다 달라지고 버그가 배포 후에야 발견됨

문제점 정의
문제 1 · 구현
플랫폼 간 구현 불일치
iOS · Android · Web이 동일 기능을 각자 방식으로 구현. 공통 로직·예외 처리 기준 없음.
→ 플랫폼별 동작 차이 · 재작업 루프
문제 2 · 품질
코드 품질 기준 부재
컨벤션, 네이밍, 아키텍처 패턴이 개발자 재량에 맡겨짐.
→ 유지보수 비용 증가 · 온보딩 난이도 상승
문제 3 · 테스트
단위·UI 테스트 자동화 부재
변경 시 어디가 깨지는지 알 수 없어 전수 수동 확인에 의존.
→ 회귀 버그 상존 · 핫픽스 반복
해결 방안
문제 1 · 구현 불일치
기획서 기반 플랫폼 공통 비즈니스 로직 명세 및 엣지케이스 자동 정리
구현 전 공통 기준 확보 → 플랫폼 간 불일치 사전 차단
문제 2 · 품질 기준 부재
코드 컨벤션·아키텍처 패턴 준수 여부 자동 검토 및 리뷰
개발자 역량과 무관하게 일정 수준 이상 품질 보장
문제 3 · 테스트 부재
구현 코드와 기획서 기반 단위·UI 테스트 자동 생성
기능 추가·수정 시 회귀 버그 자동 감지

2. 최적화

타 부서 단독 또는 협업 작업
기획 회의 · 개발 회의 · 통합 테스트

근본 원인
⚠ Root Cause

"기획서를 각 직군이 독립적으로 해석하고 작업을 시작"
공통 해석 기준 없이 백엔드·디자인·iOS·Android·Web이 동시 작업 → 불일치가 구조적으로 발생

문제점 정의
문제 1 · 기획
기획서 품질 문제
모호한 정책, 누락된 엣지케이스, 미정의 예외처리로 각자 빈칸을 채워 해석.
→ 직군별 해석 차이 발생
문제 2 · 디자인
누락 및 불일치 디자인
모호한 기획서 기반 디자인으로 엣지케이스 대응 및 공통화 미준수.
→ 개발자로부터 수정 요청 반복
문제 3 · 개발
공통 설계 기준 부재
용어·데이터 모델·비즈니스 로직을 누가 어떻게 해석했는지 알 수 없음.
→ 백엔드/iOS/Android/Web 다른 구현
문제 4 · 백엔드
API 스펙 검증 지연
표준 포맷 없이 작업 완료 후 Swagger 공유 → 프론트엔드 연동 전 스펙 미확인.
→ 연동 시 스펙 불일치 재작업
문제 5 · 테스트
미흡한 테스트
작업 지연으로 테스트 기간 단축, 미흡한 시나리오로 엣지케이스 미검증.
→ 배포 후 버그 발견 · 핫픽스
해결 방안
문제 1 · 기획 품질
기획서 검증 및 기획회의용 체크리스트 자동 작성
회의 전 기획서 완성도 확보 → 해석 차이 원천 차단
문제 2 · 디자인 불일치
기획서 기반 UI 흐름도 자동 산출
누락 엣지케이스 대응 및 화면 전환 통일화
문제 3 · 설계 기준 문제 4 · API 스펙
기획서 기반 데이터 흐름도 및 공통 설계서 스캐폴드 산출
개발회의 기반 작업 범위·API 엔드포인트 사전 확정
문제 5 · 미흡한 테스트
기획서·설계서·수정 코드 기반 테스트 시나리오 자동 작성
통합 테스트 QA 시나리오 완성도 확보
