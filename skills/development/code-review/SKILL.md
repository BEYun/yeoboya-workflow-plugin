# code-review

구현된 코드를 공통 설계와 CLAUDE.md 기준으로 리뷰한다. Claude 자동 리뷰 → 사용자 최종 확인 흐름.

리뷰 진행 전에 `superpowers:requesting-code-review` 스킬을 Skill 도구로 호출하여 리뷰 리거와 체크리스트를 확보한다. 해당 스킬의 가이드에 따라 리뷰를 수행한 뒤 아래 4가지 관점으로 결과를 정리한다.

---

## 입력

1. **리뷰 대상** — 구현된 코드 파일/디렉토리 (implement 스킬에서 자동 전달되거나 사용자가 경로 지정)
2. **공통 설계** — Notion에서 notion-writer 스킬로 "공통 설계" 서브페이지를 읽는다
3. **CLAUDE.md** — 프로젝트 컨벤션 기준

공통 설계가 Notion에 없으면 코드 자체 기준으로 검토하고 사용자에게 안내한다.

---

## 4가지 리뷰 관점

### (1) 설계 완전성 (vs 공통 설계)
- 설계에 명시된 모든 항목이 구현되었는가
- 모든 엣지 케이스가 처리되었는가
- 누락된 기능이 없는가

### (2) CLAUDE.md 컨벤션 준수
- 네이밍 컨벤션
- 파일/디렉토리 구조
- 아키텍처 패턴 (Clean Architecture, MVVM 등)
- DI 방식

### (3) 코드 품질
- 중복 코드
- 불필요한 복잡도
- 에러 핸들링 적절성
- 추상화 수준 일관성

### (4) 테스트 커버리지
- 비즈니스 로직이 테스트되고 있는가
- 엣지 케이스가 테스트되고 있는가
- 테스트가 동작(behavior)을 검증하는가 (Mock 인터랙션이 아닌)

---

## 결과 포맷

```
## 코드리뷰 결과

### ✓ 통과 항목
- [설계 완전성] 모든 화면과 상태가 구현됨
- [컨벤션] MVVM 패턴 준수

### ⚠ 개선 권장
- [코드 품질] ProfileViewModel.swift:45 — 중복 로직, 메서드 추출 권장
- [테스트] 네트워크 에러 케이스 테스트 누락

### ✗ 수정 필요
- [설계 불일치] 빈 목록 상태 UI 미구현 (공통 설계 6번 항목)

수정 후 다시 리뷰하시겠습니까?
```

---

## 리뷰 루프

✗ 수정 필요 항목이 있으면 리뷰를 종료하지 않는다:

```
수정 필요 항목 존재 → 사용자 수정 → 재리뷰
모든 항목 통과 → 리뷰 완료
```

모든 ✗ 항목이 해결될 때까지 반복한다.

---

## 리뷰 통과 시

```
✓ 코드리뷰 통과! 수정 필요 항목이 없습니다.
⚠ 개선 권장 항목이 N건 있습니다. 참고하여 반영 여부를 결정하세요.
```
