# notion-writer

/dev 파이프라인의 모든 Notion 작업을 처리하는 공통 스킬. 다른 스킬이 Notion에 뭔가를 읽거나 쓸 때 이 스킬의 규칙을 따른다.

---

## Notion MCP 도구

Notion MCP 서버의 도구를 사용한다:

| 도구 | 용도 |
|------|------|
| search | 작업번호 등으로 페이지 검색 |
| fetch | 페이지 ID로 내용 조회 |
| create-pages | 부모 페이지 하위에 서브페이지 생성 |
| update-page | 기존 페이지 내용 수정 |

---

## 공통 규칙

1. **담당자 필수** — 모든 서브페이지에 담당자 필드를 넣는다. 값은 `.claude/dev-config.json`의 `worker`.

2. **중복 생성 금지** — 같은 부모 아래 같은 제목의 서브페이지가 이미 있으면 새로 만들지 않고 기존 페이지를 업데이트한다.

3. **회의 기록 보존** — UI 흐름도와 데이터 흐름도 페이지 하단에는 개발회의용 섹션(논의사항, 수정사항)이 있다. 산출물 내용을 업데이트할 때 이 회의 기록 영역은 건드리지 않는다.

4. **읽고 나서 쓰기** — 기존 페이지를 업데이트할 때는 먼저 fetch로 현재 내용을 읽고, 산출물 영역만 교체한다.

---

## 작업번호 페이지 조회/생성

```
1. skills/setting/service-config 스킬에서 현재 서비스의 정책서/설계서 페이지 ID 조회 (매핑 키: design)
2. search로 해당 페이지 하위에서 작업번호 검색
3. 있으면 → 페이지 ID 반환
4. 없으면 → create-pages로 작업번호를 제목으로 새 페이지 생성 후 ID 반환
```

---

## 서브페이지 완료 여부 확인

산출물 기반으로 각 단계의 완료 상태를 확인한다:

| 단계 | 확인할 서브페이지 제목 | 위치 |
|------|----------------------|------|
| 1. 기획서 검토 | 기획서 검토 | 정책서/설계서 페이지 > 작업번호 |
| 2. UI 흐름도 | UI 흐름도 | 정책서/설계서 페이지 > 작업번호 |
| 3. 데이터 흐름도 | 데이터 흐름도 | 정책서/설계서 페이지 > 작업번호 |
| 4/5. 공통 설계 | 공통 설계 | 정책서/설계서 페이지 > 작업번호 |
| 6. 버그 분석 | 버그 분석 리포트 | 정책서/설계서 페이지 > 작업번호 |
| 7. QA 시트 | QA 시트 | QA 보드 페이지 > 작업번호 |

확인 방법: 작업번호 페이지의 하위에서 해당 제목의 서브페이지가 존재하면 완료.

---

## 서브페이지 생성/업데이트

```
1. 작업번호 페이지 ID 확보 (위 흐름 참조)
2. 해당 제목의 서브페이지 검색
3. 없으면:
   - create-pages로 새 서브페이지 생성
   - 담당자 필드 포함
   - 전체 내용 작성
4. 있으면:
   - fetch로 기존 내용 읽기
   - 산출물 영역만 새 내용으로 교체
   - 회의 기록 영역(논의사항, 수정사항)은 그대로 유지
```

---

## QA 보드 저장 (7단계)

```
1. skills/setting/service-config에서 현재 서비스의 qa 페이지 ID 조회
2. qa 페이지 하위에서 작업번호 페이지 검색/생성
3. 작업번호 페이지 하위에 "QA 시트" 서브페이지 생성/업데이트
```

## Notion 페이지 구조

```
[정책서/설계서 페이지] (design)
  └── [작업번호]/
      ├── 기획서 검토          ← 1단계
      ├── UI 흐름도           ← 2단계
      │   ├── (산출물 영역)
      │   ├── 논의사항         ← 개발회의 기록
      │   └── 수정사항         ← 개발회의 결과
      ├── 데이터 흐름도        ← 3단계
      │   ├── (산출물 영역)
      │   ├── 논의사항
      │   └── 수정사항
      ├── 공통 설계            ← 4/5단계
      └── 버그 분석 리포트      ← 6단계 (범위 밖 버그)

[QA 보드 페이지] (qa)
  └── [작업번호]/
      └── QA 시트             ← 7단계
```
