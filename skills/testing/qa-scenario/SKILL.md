# qa-scenario

기획서, 공통 설계, 구현 코드를 종합하여 QA용 테스트 시나리오를 작성한다. Notion QA 보드에 저장하여 QA 팀이 바로 사용할 수 있게 한다.

---

## 선행조건

4단계(기능 신규 개발) 또는 5단계(기능 변경)가 완료되어야 한다. git log에서 `[작업번호]` 패턴 커밋이 존재하는지 확인한다.

---

## 입력

1. **기획서 PDF** — 기능 요구사항 원본
2. **공통 설계** — Notion에서 **skills/common/notion-writer** 스킬로 "공통 설계" 서브페이지를 읽는다
3. **구현 코드** — 현재 프로젝트의 코드

---

## 3단계 테스트 케이스 도출

### (1) 기획서 기반 → 정상 케이스

기획서의 기능 요구사항에서 정상 시나리오를 도출한다. 각 화면의 핵심 기능별 1개 이상.

### (2) 공통 설계 기반 → 엣지 케이스

공통 설계의 상태 정의와 엣지 케이스 처리 섹션에서 도출한다. 에러 상태, 빈 상태, 권한 거부, 네트워크 오류 등.

### (3) 구현 코드 기반 → 회귀 케이스

구현 코드의 분기 조건(if/else, switch, guard, try/catch)을 분석하여 누락된 경로를 보완한다.

## 사용자 확인

테스트 시나리오를 사용자에게 보여주고 확인을 받는다:
```
정상 N건, 엣지 M건, 회귀 K건 — 총 X건의 테스트 시나리오입니다.
Notion QA 보드에 저장하시겠습니까?
```

---

## Notion 저장

**skills/common/notion-writer** 스킬을 사용한다. **QA 보드 페이지**에 저장한다 (정책서/설계서 페이지가 아님):

1. service-config에서 **qa** 페이지 ID 조회
2. qa 페이지 하위에서 작업번호 페이지 검색/생성
3. 작업번호 페이지 하위에 **"QA 시트"** 서브페이지 생성

서브페이지 구조:
```
[작업번호] QA 시트
├── 담당자: [worker]
├── 플랫폼: [platform]
├── 정상 케이스 (테이블)
├── 엣지 케이스 (테이블)
└── 회귀 케이스 (테이블)
```
