# task-complete

작업의 코드 구현과 QA가 모두 완료된 후, git 브랜치 머지부터 상태 정리까지 작업 종료 절차를 수행한다.

---

## 입력

- `.claude/active-task`에서 작업번호를 읽는다
- `dev-config.json`에서 `platform`, `worker`를 읽는다

---

## 1. 머지 대상 브랜치 결정

**CLAUDE.md 또는 프로젝트 룰에 배포 브랜치가 정의된 경우** → 그대로 사용한다. 사용자에게 확인만 받는다:
```
CLAUDE.md 기준 머지 대상: <브랜치명>
이 브랜치로 머지할까요? (y/n)
```

**정의되지 않은 경우** → 현재 로컬 브랜치 목록을 표시하고 선택받는다:
```
현재 브랜치: <작업브랜치>
머지할 대상 브랜치를 선택하세요:
  1. main
  2. develop
  ...
```

그리고 아래 권장 메시지를 출력한다:
```
💡 CLAUDE.md나 프로젝트 룰에 배포 브랜치를 명시해두면 매번 선택할 필요가 없습니다.
예시) ## 배포 브랜치
      - 기능 개발: develop
      - 핫픽스: main
```

---

## 2. 머지 및 브랜치 정리

선택한 대상 브랜치로 머지를 실행한다.

**충돌 발생 시:**
```
머지 충돌이 발생했습니다. 충돌을 해결한 후 다시 실행해주세요.
```
상태 변경 없이 종료.

**머지 성공 시** → 작업 브랜치 삭제 여부를 묻는다:
```
<작업브랜치> 브랜치를 삭제하시겠습니까? (y/n)
```
`y`면 로컬 브랜치 삭제, `n`이면 넘어간다.

---

## 3. Notion 완료 표시

Notion MCP로 작업번호 페이지에서 플랫폼에 해당하는 완료 체크박스 속성을 `true`로 갱신한다.

- `platform === "iOS"` → `iOS 완료` 체크박스
- `platform === "Android"` → `Android 완료` 체크박스

---

## 4. 상태 정리

1. `.dev-work/<작업번호>/` 디렉토리를 삭제한다
2. `.claude/active-task`에서 해당 작업번호를 제거한다

---

## 완료 메시지

```
✓ [작업번호] 작업이 완료되었습니다.
  - 머지: <작업브랜치> → <대상브랜치>
  - Notion: <platform> 완료 표시
  - 상태 정리 완료
```
