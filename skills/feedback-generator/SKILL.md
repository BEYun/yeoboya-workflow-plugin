# feedback-generator

기획서 검토(spec-review) 결과를 기획자에게 전달할 수 있는 피드백 문서로 만든다. 다크 에디토리얼 스타일의 HTML을 생성하고, 가능하면 PDF로 변환한다.

---

## 입력

- 작업번호 (예: DCL-1351)
- 검토 결과 데이터: 페이지별 이슈 목록 (유형, 제목, 문제점, 권장 수정사항)
- `.claude/dev-config.json`에서 작업자 이름

---

## 디자인 가이드라인

다크 에디토리얼 스타일로 생성한다. 기획자가 받아보고 한눈에 파악할 수 있어야 한다.

**컬러 시스템:**
- 배경: `#0a0a0b` (bg), `#141416` (surface), `#1c1c20` (raised)
- 텍스트: `#e8e6e3` (primary), `#8a8a8f` (muted), `#5a5a60` (dim)
- 액센트: `#f0c040` (골드)
- 이슈 유형별:
  - 상태 정의: `#ff6b6b` (빨강)
  - 엣지 케이스: `#ffa94d` (주황)
  - 화면 흐름: `#74c0fc` (파랑)
  - 인터랙션: `#da77f2` (보라)
- 통과: `#51cf66` (초록)

**폰트:**
- Google Fonts: `Noto Sans KR` (본문), `Playfair Display` (숫자/타이틀), `JetBrains Mono` (라벨/태그)

**레이아웃 구조:**
```
[Hero] — 작업번호, 문서 제목, 검토자/검토일
[Stats Bar] — 이슈 수, 통과 수, 검토 페이지 수, 이슈 발견율 (4칸 그리드)
[Page Sections] — 페이지별 카드
  [Page Header] — 페이지 번호 + 제목 + 이슈 건수 배지
  [Issue Cards] — 왼쪽 색상 바 + 유형 태그 + 제목 + 문제점 + 권장사항
  또는 [Pass Card] — 초록 체크 + 통과 메시지
[Summary] — 이슈 유형별 분포 (4칸 그리드, 진행 바)
[Footer] — Generated by /dev spec-review
```

**이슈 카드 구조:**
```html
<div class="issue critical"> <!-- critical/warn/info 중 택 1 -->
  <div class="issue-top">
    <span class="issue-tag tag-state">상태 정의</span> <!-- tag-state/tag-edge/tag-flow/tag-interaction -->
    <span class="issue-title">이슈 제목</span>
  </div>
  <div class="issue-problem">문제점 설명</div>
  <div class="issue-suggestion"><strong>권장 →</strong> 수정 제안</div>
</div>
```

**이슈 유형 → 태그/색상 매핑:**

| 검토 관점 | CSS class | 카드 border | 태그 색상 |
|-----------|-----------|-------------|----------|
| 상태 정의 | `critical`, `tag-state` | 빨강 | 빨강 |
| 엣지 케이스 | `warn`, `tag-edge` | 주황 | 주황 |
| 화면 흐름 | `info`, `tag-flow` | 파랑 | 파랑 |
| 인터랙션 | `info`, `tag-interaction` | 보라 | 보라 |

**통과 카드:**
```html
<div class="pass-card">
  <div class="pass-icon">✓</div>
  검토 통과 — [통과 사유]
</div>
```

## HTML 생성

1. `.claude/dev-config.json`에서 작업자 이름을 읽는다
2. 위 디자인 가이드라인에 따라 전체 HTML을 생성한다
   - Hero에 작업번호, 문서 제목, 검토자, 검토일 채우기
   - Stats Bar에 이슈 수, 통과 수, 페이지 수, 발견율 계산
   - 페이지별로 이슈 카드 또는 통과 카드 생성
   - Summary에 유형별 분포 집계
3. `.dev-work/[작업번호]/[작업번호]-feedback.html`로 저장한다

## PDF 변환

HTML 저장 후 PDF 변환을 시도한다. 우선순위:

**1순위: puppeteer**
```bash
node -e "
const puppeteer = require('puppeteer');
(async () => {
  const browser = await puppeteer.launch();
  const page = await browser.newPage();
  await page.setContent(require('fs').readFileSync('[HTML 경로]', 'utf8'));
  await page.pdf({ path: '[PDF 경로]', format: 'A4', printBackground: true });
  await browser.close();
})();
"
```

**2순위: wkhtmltopdf**
```bash
wkhtmltopdf [HTML 경로] [PDF 경로]
```

**3순위: 변환 도구 없음**
```
PDF 변환 도구를 찾을 수 없습니다.
HTML 파일이 생성되었습니다: .dev-work/[작업번호]/[작업번호]-feedback.html
브라우저에서 열어 인쇄(Cmd+P)로 PDF 변환할 수 있습니다.

자동 변환을 위해 다음 중 하나를 설치하세요:
- npm install -g puppeteer
- brew install wkhtmltopdf
```
