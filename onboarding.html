SECTION 01

개요

1
초기 설정 /dev-init을 실행해 서비스·플랫폼·작업자 정보와 Notion MCP 연결을 설정한다. 정보는 .claude/dev-config.json에 저장된다.
2
파이프라인 진입 /dev [일감번호]를 실행하면 .claude/active-task에 일감번호가 기록되고, 단계별 스킬이 순서대로 라우팅된다.
3
상태 추적 각 단계 완료 시 .dev-work/[일감번호]/state.json이 갱신된다. Hook이 Notion 쓰기를 실시간 검증해 잘못된 순서의 산출물 작성을 자동 차단한다.

4.2 Hook 파이프라인
PreToolUse stage-guard.js

Notion 쓰기(notion-create-pages, notion-update-page) 직전에 실행. 현재 파이프라인 단계와 산출물 제목이 일치하는지 검증합니다.

  • active-task가 비어 있으면 허용
  • 알 수 없는 페이지 제목이면 허용
  • 단계 불일치 시 쓰기 차단
  • DEV_GUARD_BYPASS=1 으로 우회 가능
PreToolUse validate-guard.js

산출물 저장 전 단계별 포맷 규칙을 검증. 필수 섹션 누락·형식 오류 시 쓰기를 차단합니다.

  • 단계별 검증 규칙 독립 적용
  • 규칙 미충족 시 오류 메시지와 함께 차단
  • validate-guard.js 직접 편집으로 규칙 수정 가능
PostToolUse pageid-capture.js

Notion 쓰기 성공 후 응답에서 page_id를 자동 추출해 state.json에 기록합니다.

  • 응답에서 page_id 추출
  • stages[N.M].artifactPageId 에 저장
  • stages[N.M].validated 를 true로 갱신

.dev-work/DCL-1351/state.json
{ "task": "DCL-1351", "workType": "new", "finalSpec": null, "activeStage": "3.1", "activeSkill": "skills/blueprint/ui-flow", "stages": { "1.1": { "done": false, "at": null }, "2.1": { "done": false, "validated": false, "artifactPageId": null }, "2.2": { "done": false, "validated": false, "artifactPageId": null }, "3.1": { "done": false, "validated": false, "artifactPageId": null }, "3.2": { "done": false, "validated": false, "artifactPageId": null }, "3.3": { "done": false, "validated": false, "artifactPageId": null }, "4.1": { "done": false, "result": null }, "4.2": { "done": false }, "4.3": { "done": false }, "5.1": { "done": false, "validated": false, "artifactPageId": null } }, "lastUpdated": "2026-04-21T09:00:00.000Z" }
task현재 작업의 일감번호. /dev [일감번호] 실행 시 설정됩니다.
workTypework-define(1.1)에서 선택한 작업 종류. new / change / bugfix 중 하나.
finalSpecspec-finalize(2.2) 완료 후 Notion 최종 기획서의 page_id가 저장됩니다.
activeStage현재 진행 중인 단계 번호. Hook이 이 값을 기준으로 단계 검증을 수행합니다.
activeSkill현재 실행 중인 스킬 경로. 디버깅 시 어느 스킬이 동작 중인지 파악할 수 있습니다.
stages[N.M].done해당 단계 완료 여부. true가 되면 다음 단계 진행이 허용됩니다.
stages[N.M].validatedNotion 산출물이 있는 단계에서 pageid-capture Hook이 page_id를 캡처했을 때 true로 전환됩니다.
stages[N.M].artifactPageId해당 단계의 Notion 산출물 페이지 ID. pageid-capture Hook이 자동으로 채웁니다.
stages[4.1].resultdesign-check(4.1) 결과값. 디자인 시스템 연결 여부를 code-write가 참조합니다.
lastUpdatedstate.json이 마지막으로 갱신된 ISO 8601 타임스탬프.
4.3.2 .claude/active-task

현재 /dev가 작업 중인 일감번호 하나만 기록된 텍스트 파일입니다. Hook은 이 파일을 읽어 어떤 state.json을 참조할지 결정합니다.

/dev DCL-1351 실행 active-task에 DCL-1351 기록. 이후 Hook은 .dev-work/DCL-1351/state.json 참조.
다른 일감 전환: /dev DCL-2000 active-task가 DCL-2000으로 교체. 기존 DCL-1351의 state.json은 그대로 보존됩니다.
원래 일감 복귀: /dev DCL-1351 재실행 active-task가 다시 DCL-1351로 전환. 중단된 시점의 상태에서 이어서 진행합니다.
active-task가 비어 있는 경우 Hook이 단계 검증을 건너뛰고 모든 Notion 쓰기를 허용합니다.
